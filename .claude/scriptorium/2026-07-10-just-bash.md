# just_bash

**URL:** https://github.com/elixir-ai-tools/just_bash
**Verdict: evaluate** — no current use in Faber (skip there), but the right first candidate
whenever an Elixir app wants to hand an LLM/agent a *safe* Bash tool.

## What it is

Pure-Elixir bash interpreter (v0.3.0, April 2026, ~91 stars): parses shell to an AST and
executes against an in-memory virtual filesystem (`vfs` dep). No real processes, real FS, or
network by default (opt-in curl/wget behind HTTPS-only + host allowlists). ~80 builtins incl.
`grep`/`sed`/`awk`/`jq`; pipes, redirections, functions, arrays, arithmetic, globs. Immutable
API: `bash = JustBash.new(); {result, bash} = JustBash.exec(bash, cmd)` → `stdout`/`stderr`/
`exit_code` + threaded env. "Custom commands" register trusted Elixir functions as shell
commands — note they **bypass the sandbox entirely** (host-side extensions, not sandboxed).

## When to reach for it

- Giving an AI agent a Bash tool (e.g. an MCP tool or in-app agent runtime) where the commands
  are LLM-generated and must not touch the host — this is its designed use case.
- Testing shell-ish logic hermetically (no system deps, no tmp dirs).
- Multi-tenant scenarios needing per-tenant isolated "shells".

## When NOT to

- Anything that must run a real external binary (`git`, `python3`, `sqlite3`, `claude` …) — an
  in-memory interpreter cannot exec host programs. Use `System.cmd`+timeout wrapper or `exile`
  (per [[Faber Elixir Tooling Decisions]]).
- Performance-critical shelling — it's an interpreter, slower than native bash.
- As a security boundary *with* custom commands registered — those run arbitrary Elixir
  unsandboxed, so the boundary is only as strong as the commands you add.

## Faber assessment (2026-07-10)

Skipped: Faber never executes untrusted shell (LLM output is structured JSON; transcript bash
is only *read*), and all four real shell-out sites need real binaries. Two dormant angles —
AST-accurate transcript command classification, and syntax-checking generated SKILL.md bash
examples in the eval — logged in `faber/.claude/research/2026-07-10-just-bash-evaluation.md`.
